## 第一章：项目概述

### 1.1 什么是 Jimi

Jimi 是一个功能强大的 AI 驱动的命令行智能代理系统，专为软件开发场景设计。它将大语言模型
（LLM）的智能能力与丰富的开发工具生态系统完美结合，为软件开发者提供智能化的开发助手体验。

### 1.2 核心价值

- **🚀 提升开发效率**：通过 AI 自动化完成重复性工作，让开发者专注于创造性任务
- **🧠 智能决策支持**：提供即时的技术咨询、代码建议和最佳实践指导
- **🔧 工具集成平台**：统一的工具调用接口，支持文件操作、Shell 命令、网络访问等
- **🎯 专业化分工**：多 Agent 协作机制，不同 Agent 专注不同领域（设计、编码、审查等）
- **📚 知识注入系统**：Skills 功能支持领域知识的模块化注入和自动激活

### 1.3 适用场景

Jimi 可以帮助你完成以下任务：

- **代码开发**：代码生成、重构、优化
- **问题诊断**：技术问题诊断与解决方案建议
- **代码审查**：代码质量分析、安全审计、最佳实践检查
- **自动化任务**：批量文件操作、脚本执行
- **架构设计**：项目架构设计与咨询
- **文档生成**：技术文档编写与维护
- **测试开发**：测试用例生成、测试执行与分析
- **项目构建**：编译、依赖管理、错误修复

### 1.4 系统特性

#### 🤖 CLI 智能代理
- 自然语言交互界面
- 交互式 Shell 环境（基于 JLine）
- 丰富的元命令支持
- 彩色输出与中文友好的自动换行

#### 🎭 多 Agent 支持
- **默认 Agent**：通用开发助手
- **设计 Agent**：需求分析、架构设计、技术选型
- **编码 Agent**：代码实现、重构、优化
- **审查 Agent**：代码质量审查、安全审计
- **构建 Agent**：项目构建、依赖管理
- **测试 Agent**：测试运行与分析
- **调试 Agent**：错误诊断与修复
- **文档 Agent**：文档编写与维护
- **研究 Agent**：技术信息搜索与研究
- **自定义 Agent**：支持 YAML 配置自定义 Agent

#### 🧩 Skills 系统
- 领域知识模块化管理
- 关键词自动匹配激活
- 团队知识共享与复用
- 高性能缓存机制（Caffeine）

#### 🔌 LLM 集成
- 支持多种 LLM 提供商（OpenAI、Kimi、Qwen、DeepSeek、Ollama、Claude）
- 灵活的模型切换
- 流式响应处理
- 上下文智能压缩

#### 🌐 MCP 协议支持
- Model Context Protocol 标准集成
- HTTP 和 StdIO 两种通信方式
- 外部工具服务集成
- 示例配置（数据库、文件系统、Git、GitHub 等）

#### 🛠️ 丰富的工具系统
- **文件操作**：读取、写入、搜索、补丁应用
- **Shell 执行**：命令执行与结果捕获
- **网络工具**：网页内容抓取、HTTP 请求
- **待办事项**：任务管理与追踪
- **思考工具**：复杂推理与计划
- **子 Agent 任务委托**：复杂任务分解与并行处理
- **MCP 工具**：动态加载外部工具

#### 🔐 高级功能
- **上下文压缩**：智能消息压缩，节省 Token
- **审批机制**：敏感操作的人工审批控制
- **YOLO 模式**：自动批准所有操作，提升效率
- **循环控制**：最大步数与运行时间限制
- **会话管理**：会话持久化与恢复

```
                  │
┌─────────────────▼───────────────────┐
```

## 第三章：安装配置

### 3.1 环境要求

#### 系统要求
- **操作系统**：macOS、Linux 或 Windows
- **Java**：17 或更高版本
- **Maven**：3.6 或更高版本
- **磁盘空间**：至少 500MB
- **内存**：建议 2GB 以上

#### 可选依赖
- **Git**：用于克隆项目
- **curl/wget**：用于下载资源

### 3.2 安装方式

Jimi 提供两种安装方式：

#### 方式一：一键安装（推荐）

一键安装脚本会自动完成所有安装步骤：

```bash
cd /path/to/Jimi
./scripts/quick-install.sh
```

安装过程包括：
1. 环境检查
2. 依赖安装（如需要）
3. 项目构建
4. 配置初始化
5. 启动服务

#### 方式二：分步安装

如果你想更精细地控制安装过程：

**步骤 1：检查环境**
```bash
./scripts/check-env.sh
```

这会检查：
- 操作系统类型
- Java 版本（需要 Java 17+）
- Maven 版本
- Git 工具
- 磁盘空间和内存
- 网络连接

**步骤 2：安装依赖（如需要）**
```bash
./scripts/install-deps.sh
```

支持的系统：
- macOS (使用 Homebrew)
- Ubuntu/Debian (使用 APT)
- CentOS/RHEL (使用 YUM/DNF)

**步骤 3：构建项目**
```bash
# 跳过测试快速构建
./scripts/build.sh --skip-tests

# 完整构建（包含测试）
./scripts/build.sh
```

**步骤 4：初始化配置**
```bash
./scripts/init-config.sh
```

配置向导会引导你完成：
- LLM 服务提供商选择
- API Key 配置
- 模型选择
- 高级参数设置

**步骤 5：启动 Jimi**
```bash
./scripts/start.sh
```

### 3.3 配置详解

#### 配置文件位置

Jimi 的配置文件位于：`~/.jimi/config.json`

#### 配置文件结构

```json
{
  "default_model": "qwen-max",
  "providers": {
    "qwen": {
      "type": "qwen",
      "base_url": "https://dashscope.aliyuncs.com/compatible-mode/v1",
      "api_key": "your-api-key-here",
      "custom_headers": {}
    }
  },
  "models": {
    "qwen-max": {
      "provider": "qwen",
      "model": "qwen-max",
      "max_context_size": 200000
    }
  },
  "loop_control": {
    "max_steps_per_run": 30,
    "max_retries_per_step": 10
  }
}
```

#### 配置项说明

**default_model**
- 类型：字符串
- 说明：默认使用的模型名称
- 示例：`"qwen-max"`

**providers**
- 类型：对象
- 说明：LLM 提供商配置
- 支持的提供商：
  - `kimi`：Moonshot AI
  - `qwen`：阿里云通义千问
  - `deepseek`：DeepSeek
  - `ollama`：本地 Ollama
  - `openai`：OpenAI
  - `claude`：Anthropic Claude

**models**
- 类型：对象
- 说明：模型配置
- 字段：
  - `provider`：提供商名称
  - `model`：模型名称
  - `max_context_size`：最大上下文大小

**loop_control**
- 类型：对象
- 说明：循环控制配置
- 字段：
  - `max_steps_per_run`：每次运行的最大步数
  - `max_retries_per_step`：每步的最大重试次数


- `max_retries_per_step`：每步工具调用失败时的最大重试次数

## 第五章：核心功能

### 5.1 多 Agent 系统

Jimi 提供了多个专业化的 Agent，每个 Agent 专注于特定领域的任务。

#### 可用的 Agent

**Default Agent（默认 Agent）**
- 用途：通用开发助手
- 工具：所有工具
- 特点：全能型，适合大多数场景

**Design Agent（设计 Agent）**
- 用途：需求分析、架构设计、技术选型
- 工具：Think、ReadFile、WriteFile、Grep、Glob、FetchURL
- 特点：擅长系统设计和技术方案规划

**Code Agent（编码 Agent）**
- 用途：代码实现、重构、优化
- 工具：ReadFile、WriteFile、StrReplaceFile、PatchFile、Grep、Glob、Think
- 特点：专注于代码编写和修改

**Review Agent（审查 Agent）**
- 用途：代码质量审查、最佳实践检查
- 工具：ReadFile、Grep、Glob、Think
- 特点：只读模式，不修改代码

**Build Agent（构建 Agent）**
- 用途：项目构建、依赖管理、编译错误修复
- 工具：Bash、ReadFile、WriteFile、Grep、Think
- 特点：擅长处理构建和编译问题

**Test Agent（测试 Agent）**
- 用途：测试运行与分析
- 工具：Bash、ReadFile、WriteFile、Grep、Think
- 特点：专注于测试执行和失败分析

**Debug Agent（调试 Agent）**
- 用途：错误诊断与修复
- 工具：Bash、ReadFile、WriteFile、StrReplaceFile、Grep、Glob、Think
- 特点：擅长调试和修复运行时错误

**Doc Agent（文档 Agent）**
- 用途：技术文档编写与维护
- 工具：ReadFile、WriteFile、Grep、Glob、Think、FetchURL
- 特点：专注于文档生成

**Research Agent（研究 Agent）**
- 用途：搜索和研究技术信息
- 工具：FetchURL、SearchWeb、Think
- 特点：擅长网络搜索和信息整理

### 7.5 开发新命令

#### 步骤 1：创建 CommandHandler

```java
package io.leavesfly.jimi.command.handlers;

import io.leavesfly.jimi.command.CommandContext;
import io.leavesfly.jimi.command.CommandHandler;
import org.springframework.stereotype.Component;
import java.util.List;

@Component
public class CustomCommandHandler implements CommandHandler {
    
    @Override
    public String getName() {
        return "custom";
    }
    
    @Override
    public String getDescription() {
        return "自定义命令";
    }
    
    @Override
    public List<String> getAliases() {
        return List.of("cust", "c");
    }
    
    @Override
    public String getUsage() {
        return "/custom [arg]";
    }
    
    @Override
    public void execute(CommandContext context) throws Exception {
        String arg = context.getArg(0);
        context.getOutputFormatter()
            .printSuccess("执行自定义命令: " + arg);
    }
}
```

## 第八章：故障排除

### 8.1 常见问题

#### 问题 1：Java 版本过低

**症状**
```
Java 版本过低，需要 Java 17+，当前版本: 8
```

**解决方案**

macOS:
```bash
brew install openjdk@17
export JAVA_HOME=$(/usr/libexec/java_home -v 17)
```

Linux (Ubuntu/Debian):
```bash
sudo apt install openjdk-17-jdk
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
```

将 export 命令添加到 `~/.zshrc` 或 `~/.bash_profile`。

#### 问题 2：未找到 JAR 文件

**症状**
```
未找到 Jimi JAR 文件
请先运行构建脚本: ./scripts/build.sh
```

**解决方案**
```bash
./scripts/build.sh --skip-tests
```

#### 问题 3：API Key 无效

**症状**
```
LLM API 调用失败: 401 Unauthorized
```

**解决方案**

检查配置文件 `~/.jimi/config.json`：
```json
{
  "providers": {
    "qwen": {
      "api_key": "your-valid-api-key"
    }
  }
}
```

或重新运行配置向导：
```bash
./scripts/init-config.sh
```

#### 问题 4：构建失败

**症状**
```
Maven 构建失败
```

**解决方案**

清理后重试：
```bash
./scripts/build.sh --clean
```

检查网络连接：
```bash
ping repo.maven.apache.org
```

使用国内镜像（在 `~/.m2/settings.xml` 中）：
```xml
<mirror>
  <id>aliyun</id>
  <mirrorOf>central</mirrorOf>
  <url>https://maven.aliyun.com/repository/central</url>
</mirror>
```

#### 问题 5：内存不足

**症状**
```
java.lang.OutOfMemoryError: Java heap space
```

**解决方案**

增加内存：
```bash
./scripts/start.sh -Xmx2g
```

#### 问题 6：端口冲突

**症状**
```
Address already in use
```

**解决方案**

查找占用端口的进程：
```bash
lsof -i :8080
```

杀死进程：
```bash
kill -9 <PID>
```

或使用不同的端口：
```bash
./scripts/start.sh -Dserver.port=8081
```

### 8.3 性能问题

#### 响应速度慢

**可能原因**
- LLM 服务响应慢
- 上下文过长
- 网络连接问题

**解决方案**
1. 切换到更快的模型
2. 压缩上下文：`/compact`
3. 检查网络连接

#### 内存占用高

**可能原因**
- 上下文过长
- 历史消息过多
- 缓存过大

**解决方案**
1. 定期清除上下文：`/reset`
2. 减少最大步数
3. 调整 JVM 参数

### 8.4 配置问题

**解决方案**

#### 无法找到配置文件

**症状**
```
未找到配置文件: ~/.jimi/config.json
```

**解决方案**
```bash
./scripts/init-config.sh
```

### 附录 B：工具参考

#### 文件操作工具

**ReadFile**
- 功能：读取文件内容
- 参数：
  - `path`: 文件路径（必需）
- 示例：`请读取 README.md`

**WriteFile**
- 功能：写入文件内容
- 参数：
  - `path`: 文件路径（必需）
  - `content`: 文件内容（必需）
- 示例：`请创建文件 test.txt，内容为"Hello"`

**StrReplaceFile**
- 功能：字符串替换编辑文件
- 参数：
  - `path`: 文件路径（必需）
  - `old_str`: 要替换的字符串（必需）
  - `new_str`: 新字符串（必需）
- 示例：`将 config.json 中的 localhost 替换为 example.com`

**PatchFile**
- 功能：应用补丁到文件
- 参数：
  - `path`: 文件路径（必需）
  - `patch`: 补丁内容（必需）
- 示例：`应用补丁到 UserService.java`

**Glob**
- 功能：查找文件
- 参数：
  - `pattern`: 文件模式（必需）
- 示例：`查找所有 .java 文件`

**Grep**
- 功能：搜索文件内容
- 参数：
  - `pattern`: 搜索模式（必需）
  - `path`: 搜索路径（可选）
- 示例：`搜索包含 TODO 的文件`

#### Shell 工具

**Bash**
- 功能：执行 Shell 命令
- 参数：
  - `command`: Shell 命令（必需）
  - `timeout`: 超时时间（可选，默认 30 秒）
- 示例：`执行 mvn clean package`

#### 网络工具

**FetchURL**
- 功能：获取网页内容
- 参数：
  - `url`: URL 地址（必需）
- 示例：`获取 https://example.com 的内容`

**SearchWeb**
- 功能：搜索网络
- 参数：
  - `query`: 搜索查询（必需）
- 示例：`搜索"Spring Boot 最佳实践"`

#### 其他工具

**Think**
- 功能：思考工具
- 参数：
  - `content`: 思考内容（必需）
- 示例：`思考如何优化这个算法`

**SetTodoList**
- 功能：设置待办列表
- 参数：
  - `tasks`: 任务列表（必需）
- 示例：`创建待办列表`

**Task**
- 功能：子 Agent 任务委托
- 参数：
  - `agent`: Agent 名称（必需）
  - `task`: 任务描述（必需）
- 示例：`使用设计 Agent 设计订单模块`

#### 快速切换配置

```bash
#!/bin/bash
# 保存为 ~/.jimi/switch-config.sh

CONFIG_DIR="$HOME/.jimi"

case "$1" in
  dev)
    ln -sf $CONFIG_DIR/config-dev.json $CONFIG_DIR/config.json
    echo "切换到开发环境配置"
    ;;
  prod)
    ln -sf $CONFIG_DIR/config-prod.json $CONFIG_DIR/config.json
    echo "切换到生产环境配置"
    ;;
  *)
    echo "用法: $0 {dev|prod}"
    exit 1
    ;;
esac
```

使用：
```bash
~/.jimi/switch-config.sh dev
~/.jimi/switch-config.sh prod
```
